# 1. Chapter 2：Java基础、基本数据类型、字符串

## 1.1 标识符和命名规范

### 标识符规则

Java 标识符用于表示变量、方法、类等名字。

合法规则：

- 可以由字母、数字、下划线 `_`、美元符 `$` 组成。
- 不能以数字开头。
- 不能是 Java 关键字，如 `class`、`public`、`void`。
- 不能是 `true`、`false`、`null`。
- 不能包含空格和特殊符号，如 `! @ # % & *`。

例子：

```java
int age;          // 合法
String userName;  // 合法
int _count;       // 合法
int $value;       // 合法，但一般不建议自己用 $
int 2num;         // 错误，不能数字开头
int class;        // 错误，class 是关键字
```

### 命名规范

| 类型 | 推荐写法 | 示例 |
|---|---|---|
| 类名 | 首字母大写，名词，大驼峰 | `Student`, `CarRace` |
| 接口名 | 首字母大写，常用形容词或能力名 | `Runnable`, `Comparable` |
| 方法名 | 首字母小写，动词，小驼峰 | `printInfo()`, `getName()` |
| 布尔方法 | 常以 `is` 开头 | `isEmpty()` |
| 变量名 | 首字母小写，小驼峰 | `studentName` |
| 常量名 | 全大写，下划线分隔 | `MAX_VALUE` |
| 包名 | 全小写 | `java.util` |

考试人工题可以这样写：

> Java 标识符是变量、方法、类等程序实体的名字。命名时不能使用关键字，不能以数字开头，通常类名用大驼峰，方法和变量用小驼峰，常量用全大写加下划线。

## 1.2 基本数据类型与引用数据类型

Java 的类型分两类：

1. **primitive types 基本数据类型**
2. **reference types 引用数据类型**：类、接口、数组

### 8种基本数据类型

| 类型 | 位数 | 含义 | 典型用途 |
|---|---:|---|---|
| `byte` | 8 | 整数 | 小范围整数 |
| `short` | 16 | 整数 | 较少直接用 |
| `int` | 32 | 整数 | 默认整数类型 |
| `long` | 64 | 整数 | 大整数，字面量常加 `L` |
| `float` | 32 | 浮点 | 字面量加 `F` |
| `double` | 64 | 浮点 | 默认小数类型 |
| `char` | 16 | Unicode字符 | 单个字符 |
| `boolean` | 1个逻辑值 | `true/false` | 条件判断 |

注意：

```java
char c = 'A';          // 单引号，一个字符
String s = "A";        // 双引号，字符串，不是基本类型
long x = 10000000000L; // long 字面量建议加 L
float f = 3.14F;       // float 字面量必须加 F
```

## 1.3 类型转换和运算

### 自动类型转换 widening

小范围类型可以自动转成大范围类型：

```java
int a = 10;
double b = a; // int 自动转 double
```

常见顺序大致是：

```text
byte/short/char -> int -> long -> float -> double
```

### 强制类型转换 narrowing

大范围转小范围需要强转，可能丢失精度：

```java
double x = 3.9;
int y = (int)x; // 3，小数部分直接丢掉
```

### 整数除法

```java
System.out.println(5 / 2);     // 2
System.out.println(5 / 2.0);   // 2.5
System.out.println(5 % 2);     // 1
```

### 浮点数比较

不要直接比较浮点数是否相等：

```java
double a = 0.1 + 0.2;
double b = 0.3;
if (Math.abs(a - b) < 1e-10) {
    System.out.println("equal");
}
```

## 1.4 运算符重点

| 类型 | 运算符 | 注意点 |
|---|---|---|
| 算术 | `+ - * / %` | 整数除法会截断 |
| 关系 | `> >= < <= == !=` | 对象比较不要乱用 `==` |
| 逻辑 | `&& || ! ^` | `&&`、`||` 有短路 |
| 赋值 | `= += -= *= /= %=` | `a += b` 含隐式强转 |
| 条件 | `条件 ? 值1 : 值2` | 可替代简单 if-else |
| 位运算 | `& | ^ ~ << >> >>>` | 一般不是主重点 |

短路例子：

```java
int x = 0;
if (x != 0 && 10 / x > 1) { // 后半段不会执行，不报除0异常
    System.out.println("ok");
}
```

## 1.5 选择、循环、break、continue、return

### switch 注意点

```java
switch (score / 10) {
    case 10:
    case 9:
        System.out.println("A");
        break;
    case 8:
        System.out.println("B");
        break;
    default:
        System.out.println("C");
}
```

没有 `break` 会继续执行后面的 case，这叫 fall-through。

### 三种循环

```java
while (condition) { }

do { } while (condition);

for (int i = 0; i < n; i++) { }
```

增强 for：

```java
for (int x : arr) {
    System.out.println(x);
}
```

区别：

- `break`：结束当前循环。
- `continue`：跳过本轮，进入下一轮。
- `return`：直接结束方法，有返回值的方法必须返回对应类型。

## 1.6 数组

数组长度固定，通过 `length` 获取。

```java
int[] a = new int[5];
int[] b = {1, 2, 3};
System.out.println(b.length);
```

常用工具类：`java.util.Arrays`

```java
import java.util.*;

int[] a = new int[5];

System.out.println(Arrays.toString(a));

int[] b = {1, 2, 3};

System.out.println(Arrays.equals(b, c)); // true

int[] d = Arrays.copyOf(b, 5); // [1, 2, 3, 0, 0]
Arrays.sort(b);
```

二维数组：

```java
int[][] a = new int[3][4];
for (int i = 0; i < a.length; i++) {
    for (int j = 0; j < a[i].length; j++) {
        a[i][j] = i + j;
    }
}
```

## 1.7 String 字符串

### String 是引用类型，不是基本类型

```java
String s = "Hello";
```

`String` 对象不可变 immutable。所谓修改字符串，实际通常是产生了新对象：

```java
String s = "Hello";
s = s + "World"; // 原来的 "Hello" 没被改，s 指向新字符串
```

### `==` 和 `equals()`

- `==`：比较两个引用是否指向同一个对象。
- `equals()`：比较字符串内容是否相同。

```java
String s1 = new String("Hello");
String s2 = new String("Hello");
System.out.println(s1 == s2);      // false
System.out.println(s1.equals(s2)); // true

String a = "Hello";
String b = "Hello";
System.out.println(a == b);        // true，字符串常量池
System.out.println(a.equals(b));   // true
```

考试结论：**字符串内容比较，一律优先用 `equals()`。**

### 常用方法

| 方法 | 作用 |
|---|---|
| `length()` | 返回字符串长度 |
| `charAt(i)` | 返回第 i 个字符，从0开始 |
| `substring(begin)` | 从 begin 截到末尾 |
| `substring(begin, end)` | `[begin, end)`，含头不含尾 |
| `split(regex)` | 按正则分割 |
| `equals()` | 内容比较 |
| `equalsIgnoreCase()` | 忽略大小写比较 |
| `contains()` | 是否包含 |
| `indexOf()` | 第一次出现的位置 |
| `trim()` | 去掉首尾空白 |
| `toLowerCase()` | 转小写 |
| `toUpperCase()` | 转大写 |

示例：

```java
String s = "Hello World";
System.out.println(s.substring(6));     // World
System.out.println(s.substring(0, 5));  // Hello
System.out.println(s.charAt(1));        // e
```

### split 的 limit

```java
String test = "apple,banana,orange,,grape,";
System.out.println(Arrays.toString(test.split(",", 2)));
System.out.println(Arrays.toString(test.split(",", -1)));
```

大概记：

- `limit > 0`：最多分成 limit 段。
- `limit = 0`：会丢弃末尾空字符串。
- `limit < 0`：保留末尾空字符串。

## 1.8 StringBuffer / StringBuilder

PPT讲了 `StringBuffer`，它适合频繁修改字符串。

```java
StringBuffer sb = new StringBuffer();
sb.append("Welcome to Java");
sb.append("!");
sb.insert(11, "XML and ");
sb.delete(8, 11);
sb.replace(8, 11, "HTML");
sb.setCharAt(0, 'w');
System.out.println(sb.toString());
```

常用方法：

| 方法 | 作用 |
|---|---|
| `append()` | 追加 |
| `insert()` | 插入 |
| `delete(start,end)` | 删除 `[start,end)` |
| `replace(start,end,str)` | 替换 |
| `setCharAt(index,ch)` | 修改指定字符 |
| `length()` | 当前字符数 |
| `capacity()` | 当前容量 |
| `toString()` | 转 String |

实际编程里也常用 `StringBuilder`，单线程更常见；但如果老师PPT写的是 `StringBuffer`，考试答 `StringBuffer` 更稳。

## 1.9 包装类、装箱拆箱、BigDecimal

### 包装类

| 基本类型 | 包装类 |
|---|---|
| `int` | `Integer` |
| `long` | `Long` |
| `double` | `Double` |
| `float` | `Float` |

```java
BigDecimal a = new BigDecimal("1.2");
BigDecimal b = new BigDecimal("0.4");
System.out.println(a.subtract(b)); // 0.8
```

# 2. Chapter 3：类与对象

## 2.1 类与对象

- **类 class**：创建对象的代码模板，描述共同属性和行为。
- **对象 object**：类的具体实例。
- **对象的三部分**：唯一标识 identity、状态 state、行为 behavior。
- **状态**：字段、属性、成员变量。
- **行为**：方法。

人工题模板：

> 类是对象的抽象模板，定义对象共有的属性和方法；对象是类实例化后的具体实体。对象具有唯一标识、状态和行为，状态通常由成员变量表示，行为由方法表示。

---

## 2.2 面向对象三要素

| 要素 | 含义 | Java实现 |
|---|---|---|
| 封装 Encapsulation | 把数据和操作数据的方法绑定在一起，并隐藏内部实现 | `private` 字段 + `public` 方法 |
| 继承 Inheritance | 子类获得父类已有属性和方法，实现代码复用 | `extends` |
| 多态 Polymorphism | 同一接口或父类引用调用同一方法，运行时表现不同 | 方法重写、向上转型、动态绑定、接口 |

### 封装例子

```java
class Student {
    private String name;
    private int age;

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        if (age >= 0) {
            this.age = age;
        }
    }
}
```

封装的好处：数据隐藏、高内聚、低耦合。

---

## 2.3 字段、局部变量、参数

类中变量分三类：

| 类型 | 定义位置 | 生命周期 | 是否有默认值 |
|---|---|---|---|
| 成员变量 / 字段 | 类中、方法外 | 对象创建到对象销毁 | 有默认值 |
| 静态变量 / 类变量 | 类中、方法外，用 `static` | 类加载到程序结束 | 有默认值 |
| 局部变量 | 方法、构造器或代码块中 | 进入块创建，退出块销毁 | 没有默认值，必须先赋值 |
| 参数 | 方法参数列表中 | 方法调用期间 | 由实参初始化 |

局部变量没有默认值：

```java
int x;
// System.out.println(x); // 编译错误，未初始化
```

成员变量有默认值：数字为0，boolean为false，引用为null。

## 2.4 方法与方法签名

方法结构：

```java
<访问修饰符> <返回类型> <方法名>(<参数列表>) {
    // 方法体
}
```

方法签名主要由 **方法名 + 参数类型列表** 组成，不包括返回值。

```java
void setLocation(int x, int y) {}
void setLocation(Point p) {}
```

这是方法重载 overload。

## 2.5 构造方法 constructor

构造方法用于初始化对象。

特点：

- 构造方法名必须和类名相同。
- 没有返回值，连 `void` 都不能写。
- 可以重载。
- 如果一个构造方法都没写，编译器会提供无参默认构造方法。
- 如果你写了任意构造方法，编译器就不再自动提供无参构造方法。

```java
class Car {
    private double width;
    private double height;

    public Car() {
        this(0, 0); // 调用本类另一个构造方法，必须放第一句
    }

    public Car(double width, double height) {
        this.width = width;
        this.height = height;
    }
}
```

`new Car(1.0, 2.0)` 的过程：

1. 为对象成员变量分配内存。
2. 执行构造方法。
3. 返回新对象的引用。

## 2.6 this

`this` 表示当前对象。

用法：

```java
this.name = name; // 区分成员变量和参数
this(0, 0);       // 调用本类其他构造方法，只能放构造方法第一句
this.method();    // 调用当前对象的方法
```

## 2.7 参数传递

Java 只有值传递，但要分清传递的值是什么：

- 基本类型：传的是具体值的副本，方法内修改不影响外面。
- 引用类型：传的是引用值的副本，引用指向同一个对象，所以可以修改对象内部字段。

```java
class Box {
    int value;
}

static void changePrimitive(int x) {
    x = 100;
}

static void changeObject(Box b) {
    b.value = 100;
}
```

## 2.8 static

`static` 成员属于类，不属于某个对象。

```java
class Car {
    public static int count = 0;
    private String name;

    public Car(String name) {
        this.name = name;
        count++;
    }

    public static int getCount() {
        return count;
    }
}
```

注意：

- 静态方法只能直接访问静态变量和静态方法。
- 静态方法不能直接访问实例变量、实例方法，除非先创建对象。
- 静态变量可以用 `类名.变量名` 访问。
- 静态代码块在类加载时执行一次。

```java
class A {
    static int x;
    static {
        x = 10;
    }
}
```

# 3. Chapter 4：继承、接口、多态、Object类

## 3.1 继承 extends

继承表示 is-a 关系。

```java
class Vehicle {
    public void move() {
        System.out.println("move");
    }
}

class Car extends Vehicle {
    public void honk() {
        System.out.println("beep");
    }
}
```

子类可以继承父类的 `public`、`protected` 成员；同包时还能继承默认访问成员。`private` 成员不能被子类直接访问。

```java
Car c = new Car();
c.move(); // 父类方法
```

---

## 3.2 继承和组合

| 关系 | 含义 | 例子 |
|---|---|---|
| 继承 is-a | A 是一种 B | `Car extends Vehicle` |
| 组合 has-a | A 有一个 B | `Car` 里有 `Engine engine` |

考试判断：

- 能说“子类是父类的一种”时，用继承。
- 只是“拥有某个部件”时，用组合。

```java
class Engine {
    void start() {}
}

class Car {
    private Engine engine = new Engine();
    public void start() {
        engine.start();
    }
}
```

---

## 3.3 super 与构造方法链

子类构造方法会先调用父类构造方法。

```java
class Parent {
    Parent() {
        System.out.println("Parent");
    }
}

class Child extends Parent {
    Child() {
        super();
        System.out.println("Child");
    }
}
```

注意：

- `super()` 必须是构造方法第一句。
- 如果不写，默认调用父类无参构造。
- 如果父类没有无参构造，子类必须显式 `super(参数)`。
- 子类不能继承父类构造方法。

---

## 3.4 方法重载与方法重写

| 对比项 | 重载 Overloading | 重写 Overriding |
|---|---|---|
| 发生位置 | 同一个类或父子类都可以 | 父子类之间 |
| 方法名 | 相同 | 相同 |
| 参数列表 | 必须不同 | 必须相同 |
| 返回值 | 不能只靠返回值区分 | 一般相同，或协变返回 |
| 多态类型 | 编译时多态 | 运行时多态 |

### 重载

```java
void print(int x) {}
void print(String s) {}
void print(int x, int y) {}
```

### 重写

```java
class Animal {
    public void cry() {
        System.out.println("wawa");
    }
}

class Dog extends Animal {
    @Override
    public void cry() {
        System.out.println("woof");
    }
}
```

不能被真正重写：

- `private` 方法：子类不可见。
- `static` 方法：子类重新定义叫隐藏，不是运行时重写。
- `final` 方法：禁止重写。

---

## 3.5 多态、向上转型、动态绑定

```java
Animal a = new Dog(); // 向上转型
a.cry();              // 实际调用 Dog 的 cry()
```

完整例子：

```java
class Animal {
    public void cry() {
        System.out.println("wawa");
    }
}

class Dog extends Animal {
    @Override
    public void cry() {
        System.out.println("woof");
    }
}

class Cat extends Animal {
    @Override
    public void cry() {
        System.out.println("meow");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal[] animals = {new Dog(), new Cat()};
        for (Animal a : animals) {
            a.cry();
        }
    }
}
```

多态条件：

1. 有继承或接口实现关系。
2. 子类重写父类方法。
3. 父类引用指向子类对象。
4. 运行时动态绑定到实际对象的方法。

### 成员变量没有多态

```java
class P {
    int i = 8;
    void f() { System.out.println("P"); }
}
class C extends P {
    int i = 9;
    void f() { System.out.println("C"); }
}
P p = new C();
System.out.println(p.i); // 8，变量看引用类型
p.f();                   // C，方法看实际对象
```

```java
Vehicle v = new Car();

    Car c = (Car)v;
    c.honk();
```

| 用法 | 含义 |
|---|---|

| `final int x = 10` | 变量只能赋值一次 |

## 3.8 抽象类 abstract

抽象类用于定义不完整的父类模板。

```java
abstract class Shape {
    public abstract double area();

    public void printArea() {
        System.out.println(area());
    }
}

class Rectangle extends Shape {
    private double w, h;
    public Rectangle(double w, double h) {
        this.w = w;
        this.h = h;
    }
    @Override
    public double area() {
        return w * h;
    }
}
```

注意：

- 抽象类不能直接 `new`。
- 有抽象方法的类必须声明为抽象类。
- 子类如果不实现全部抽象方法，子类也必须是抽象类。

## 3.9 接口 interface

接口是方法定义的集合，也可以有常量。

```java
interface Stack {
    boolean push(Object e);
    boolean pop();
    Object top();
    boolean isEmpty();
    void clear();
}

class ArrayStack implements Stack {
    private Object[] data = new Object[100];
    private int top = -1;

    public boolean push(Object e) {
        if (top == data.length - 1) return false;
        data[++top] = e;
        return true;
    }

    public boolean pop() {
        if (isEmpty()) return false;
        top--;
        return true;
    }

    public Object top() {
        return isEmpty() ? null : data[top];
    }

    public boolean isEmpty() {
        return top == -1;
    }

    public void clear() {
        top = -1;
    }
}
```

接口规则：

- 接口方法默认是 `public abstract`。
- 接口字段默认是 `public static final`。
- 类用 `implements` 实现接口。
- 实现类必须实现接口的所有抽象方法。
- Java 类只能单继承，但可以实现多个接口。

```java
class MyClass implements InterfaceA, InterfaceB {
    // 实现所有方法
}
```

## 3.11 toString()

默认 `toString()` 形式大概是：

```text

```

通常要重写为可读字符串：

```java
class Car {

    private double width;
    private double height;

    public Car(String name, double width, double height) {
        this.name = name;
        this.width = width;

    public String toString() {
```

## 3.12 equals()

默认 `equals()` 本质是 `this == obj`，属于浅比较。实际常常需要按属性深比较。

规范模板：

```java
@Override
public boolean equals(Object obj) {
    if (this == obj) return true;
    if (obj == null) return false;
    if (getClass() != obj.getClass()) return false;
    Student other = (Student)obj;
    return age == other.age && Objects.equals(name, other.name);
}
```

更简洁写法：

```java
import java.util.Objects;

@Override
public boolean equals(Object obj) {
    if (this == obj) return true;
    if (!(obj instanceof Student)) return false;
    Student other = (Student)obj;
    return age == other.age && Objects.equals(name, other.name);
}
```

`equals()` 应满足：

1. 自反性：`x.equals(x)` 为 true。
2. 对称性：`x.equals(y)` 与 `y.equals(x)` 一致。
3. 传递性。
4. 一致性：对象未变时结果一致。
5. 与 null 比较返回 false。

---

## 3.13 hashCode()

规则：

- 如果两个对象 `equals()` 为 true，那么它们的 `hashCode()` 必须相等。
- 如果两个对象 `equals()` 为 false，hashCode 不要求一定不同，但应尽量减少冲突。
- 重写 `equals()` 时，通常必须一起重写 `hashCode()`。

推荐写法：

```java
@Override
public int hashCode() {
    return Objects.hash(name, age);
}
```

没有正确重写时，放进 `HashSet`、作为 `HashMap` 的 key 会出问题。

## 3.14 clone()：浅复制与深复制

浅复制：只复制对象本身，里面的引用字段仍指向同一个对象。  
深复制：对象里的引用字段也复制一份新对象。

```java
class Point implements Cloneable {
    int x, y;
    Point(int x, int y) { this.x = x; this.y = y; }

    @Override
    public Point clone() {
        try {
            return (Point)super.clone();
        } catch (CloneNotSupportedException e) {
            return null;
        }
    }
}

class Car implements Cloneable {
    Point location;
    Car(Point location) { this.location = location; }

    @Override
    public Car clone() {
        try {
            Car c = (Car)super.clone();
            c.location = location.clone(); // 深复制关键
            return c;
        } catch (CloneNotSupportedException e) {
            return null;
        }
    }
}
```

## 3.15 Comparable 与 Comparator

`TreeSet`、`Collections.sort()` 常需要比较规则。

### Comparable：类自己会比较

```java
class Student implements Comparable<Student> {
    String name;
    int score;

    public int compareTo(Student other) {
        return this.score - other.score; // 升序
    }
}
```

### Comparator：外部比较器

```java
Collections.sort(list, new Comparator<Student>() {
    public int compare(Student a, Student b) {
        return b.score - a.score; // 降序
    }
});
```

Lambda 写法：

```java
list.sort((a, b) -> b.score - a.score);
```

# 4. Chapter 5：异常处理

## 4.1 异常基本概念

异常是程序执行期间发生的事件，会打断正常执行流程。创建异常对象并交给运行时系统叫 throwing an exception。

Java可以捕获并处理异常，使程序从异常状态中恢复。

---

## 4.2 Throwable体系

```text
Object
  └── Throwable
        ├── Error
        └── Exception
              ├── RuntimeException
              │     ├── ArithmeticException
              │     ├── NullPointerException
              │     └── IndexOutOfBoundsException
              └── IOException / SQLException ...
```

| 类型 | 含义 | 是否通常处理 |
|---|---|---|
| `Error` | 严重问题，如虚拟机错误 | 一般不捕获 |
| `Exception` | 程序可处理异常 | 可以处理 |
| `RuntimeException` | 运行时异常，常是逻辑错误 | unchecked，不强制处理 |
| 其他Exception | 如 `IOException` | checked，必须处理或声明抛出 |

---

## 4.3 checked 与 unchecked

### checked exception

编译器强制你处理：

```java
public static void readFile() throws FileNotFoundException {
    FileInputStream fis = new FileInputStream("a.txt");
}
```

或者：

```java
try {
    FileInputStream fis = new FileInputStream("a.txt");
} catch (FileNotFoundException e) {
    System.out.println(e.getMessage());
}
```

### unchecked exception

`RuntimeException` 及其子类，不强制处理：

```java
int x = 10 / 0; // ArithmeticException
```

---

## 4.4 try-catch-finally

```java
try {
    // 可能出错的代码
} catch (ExceptionType e) {
    // 处理异常
} finally {
    // 无论是否异常，一般都会执行，常用于关闭资源
}
```

重点：

- 如果 try 中某句抛出异常，try 中后面的语句会被跳过。
- catch 执行完后，从整个 try-catch 结构后面继续执行。
- 不会回到异常发生点继续执行。
- 多个 catch 时，子类异常要写在父类异常前面。

```java
try {
    int[] a = null;
    System.out.println(a[0]);
} catch (NullPointerException e) {
    System.out.println("null");
} catch (Exception e) {
    System.out.println("other");
} finally {
    System.out.println("finally");
}
```

---

## 4.5 throw 与 throws

| 关键字 | 位置 | 作用 |
|---|---|---|
| `throw` | 方法体内 | 主动抛出一个异常对象 |
| `throws` | 方法声明处 | 声明这个方法可能抛出异常 |

```java
class Score {
    private int score;

    public Score(int score) throws IllegalArgumentException {
        if (score < 0 || score > 100) {
            throw new IllegalArgumentException("score out of range");
        }
        this.score = score;
    }
}
```

---

## 4.6 自定义异常

自定义 checked exception：

```java
class OutOfScoreRangeException extends Exception {
    private int score;

    public OutOfScoreRangeException(String message, int score) {
        super(message);
        this.score = score;
    }

    public int getScore() {
        return score;
    }
}

class Score {
    private int score;

    public Score(int score) throws OutOfScoreRangeException {
        if (score < 0 || score > 100) {
            throw new OutOfScoreRangeException("Out of score range: ", score);
        }
        this.score = score;
    }
}
```

使用：

```java
try {
    Score s = new Score(800);
} catch (OutOfScoreRangeException e) {
    System.out.println(e.getMessage() + e.getScore());
}
```

## 4.7 final、finally、finalize 区分

| 词 | 含义 |
|---|---|
| `final` | 关键字，修饰类/方法/变量，表示不能继承、不能重写、不能再次赋值 |
| `finally` | 异常处理结构中的代码块，常用于资源清理 |
| `finalize()` | Object类方法，垃圾回收前可能调用，不保证执行，不建议依赖 |

## 5.1 enum 枚举

## 5.2 泛型 Generics

泛型可以把类型参数化，提高类型安全，把运行时类型转换错误提前到编译期。

```java
class Box<E> {
    private E data;

    public Box(E data) {
        this.data = data;
    }

    public E getData() {
        return data;
    }
}

Box<Integer> intBox = new Box<>(42);
int x = intBox.getData(); // 不需要强转
```

### 泛型方法

```java
public static <T> T getMiddle(T[] arr) {
    return arr[arr.length / 2];
}
```

### 有界泛型

```java
class MathBox<E extends Number> {
    private E data;
    public MathBox(E data) { this.data = data; }
    public double sqrt() { return Math.sqrt(data.doubleValue()); }
}
```

`E extends Number` 表示 E 必须是 Number 或其子类。

### 通配符

```java
public static void printBox(Box<?> box) {
    System.out.println(box.getData());
}
```

重点坑：

```java
Box<Integer> a = new Box<>(1);
// Box<Number> b = a; // 错误：Box<Integer> 不是 Box<Number> 的子类
```

## 5.3 数组 vs 集合

| 对比 | 数组 | 集合 |
|---|---|---|
| 长度 | 固定 | 可增长 |
| 基本类型 | 可直接存 | 不能直接存，要用包装类 |
| 类型 | 同质 | 泛型下通常同质，原始类型可混杂但不推荐 |
| 方法 | 功能少 | 增删查遍历方法多 |
| 性能 | 简单、高效 | 更灵活 |

## 5.4 集合框架结构

```text

 ├── List<E>
```

- `List`：有序，可重复，可按下标访问。

## 5.5 List

常用实现：

| 实现 | 特点 |
|---|---|
| `ArrayList` | 底层动态数组，查询快，尾部添加快，中间插入删除慢 |
| `LinkedList` | 双向链表，插入删除较快，也可作队列 |
| `Vector` | 类似ArrayList，但同步，线程安全，较老 |

常用方法：

```java
List<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.add(1, "C");
System.out.println(list.get(0));
list.set(0, "X");
list.remove("B");
System.out.println(list.contains("A"));
System.out.println(list.indexOf("C"));
```

遍历：

```java
for (String s : list) {
    System.out.println(s);
}

Iterator<String> it = list.iterator();

    System.out.println(it.next());
}
```

## 5.6 Set

Set 不允许重复，判断重复主要依赖 `equals()`，HashSet 还依赖 `hashCode()`。

| 实现 | 特点 |
|---|---|
| `HashSet` | 哈希表，无序，增删查平均 O(1) |
| `TreeSet` | 红黑树，有序，O(log n)，元素要可比较 |
| `LinkedHashSet` | 哈希表 + 链表，按插入顺序 |

集合运算：

```java
Set<String> A = new HashSet<>(Arrays.asList("A", "B", "C"));
Set<String> B = new HashSet<>(Arrays.asList("B", "C", "D"));

Set<String> union = new HashSet<>(A);
union.addAll(B);       // 并集

Set<String> inter = new HashSet<>(A);
inter.retainAll(B);    // 交集

Set<String> diff = new HashSet<>(A);
diff.removeAll(B);     // 差集 A-B
```

TreeSet 存自定义对象：

```java
class Car implements Comparable<Car> {
    int price;
    Car(int price) { this.price = price; }

    public int compareTo(Car other) {
        return this.price - other.price;
    }
}
```

## 5.7 Queue

PPT中 Queue 例子用 `LinkedList` 实现。

```java
Queue<String> q = new LinkedList<>();
q.add("A");
q.add("B");
System.out.println(q.remove()); // A
System.out.println(q.element()); // 查看队头，不删除
```

注意：

- `remove()` 队空会抛 `NoSuchElementException`。
- `element()` 队空也会抛异常。
- 实际编程也可用 `poll()`、`peek()`，队空返回 null，更安全。

## 5.8 Map

Map 存储 key-value 键值对。一个 key 最多映射一个 value，key 不能重复。

常用方法：

```java
Map<String, Integer> map = new HashMap<>();

map.put("Geely", 12000);
System.out.println(map.get("BYD"));
System.out.println(map.containsKey("BYD"));
System.out.println(map.containsValue(15000));
map.remove("BYD");
System.out.println(map.size());
```

遍历：

```java
for (Map.Entry<String, Integer> e : map.entrySet()) {
    System.out.println(e.getKey() + " " + e.getValue());

```

| 实现 | 特点 |
|---|---|
| `HashMap` | 哈希表，无序，key需要合理实现 `equals/hashCode` |
| `TreeMap` | 按key排序，key需要可比较 |

词频统计模板：

```java

for (String w : words) {
    cnt.put(w, cnt.getOrDefault(w, 0) + 1);
}
```

## 7.1 Swing程序基本步骤

1. 创建顶层容器，通常是 `JFrame`。
2. 把组件添加到 `contentPane`，并设置布局。
3. 编写事件处理类并注册监听器。

6. 设置可见。

```java
import java.awt.FlowLayout;
import java.awt.event.*;
import javax.swing.*;

public class ButtonTest extends JFrame {
    private JButton exitButton;

    public ButtonTest() {
        exitButton = new JButton("Exit");
        exitButton.addActionListener(new ButtonExitHandler());
        getContentPane().setLayout(new FlowLayout());
        getContentPane().add(exitButton);
        setTitle("Exit JButton");
        setSize(300, 80);
        setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        setVisible(true);
    }

    private class ButtonExitHandler implements ActionListener {
        public void actionPerformed(ActionEvent e) {
            System.exit(0);
        }
    }

    public static void main(String[] args) {
        new ButtonTest();
    }
}
```

## 8.1 线程创建方式一：继承 Thread

```java
class MyThread extends Thread {
    @Override
    public void run() {
        for (int i = 1; i <= 5; i++) {
            System.out.println(getName() + " " + i);
        }
    }
}

public class Main {
    public static void main(String[] args) {
        MyThread t = new MyThread();
        t.start(); // 启动新线程
    }
}
```

## 8.2 线程创建方式二：实现 Runnable

```java
class MyTask implements Runnable {
    public void run() {
        System.out.println("running");
    }
}

public class Main {
    public static void main(String[] args) {
        Thread t = new Thread(new MyTask());
        t.start();
    }
}
```

实际更推荐 `Runnable`，因为 Java 单继承，继承 Thread 后就不能继承其他类。

---

## 8.3 start() 与 run()

| 方法 | 含义 |
|---|---|
| `start()` | 开启新线程，由JVM调用 `run()` |
| `run()` | 普通方法调用，不会开启新线程 |

```java
t.start(); // 正确开启线程
// t.run(); // 只是普通方法调用
```

同一个 Thread 对象不能 start 两次，否则 `IllegalThreadStateException`。

---

## 8.4 sleep()、join()、InterruptedException

```java
try {
    Thread.sleep(1000);
} catch (InterruptedException e) {
    e.printStackTrace();
}
```

`join()` 等待线程结束：

```java
Thread t = new Thread(new MyTask());
t.start();
try {
    t.join();
} catch (InterruptedException e) {
    e.printStackTrace();
}
System.out.println("done");
```

`InterruptedException` 是 checked exception，必须捕获或 `throws`。

---

## 8.5 synchronized 同步

多个线程共享数据时可能发生竞态条件，需要同步。

```java
class Counter {
    private int count = 0;

    public synchronized void add() {
        count++;
    }

    public int getCount() {
        return count;
    }
}
```

同步代码块：

```java
synchronized (this) {
    count++;
}
```

---

## 8.6 wait、notify、notifyAll

这些方法来自 `Object`，必须在同步代码块或同步方法中调用。

```java
synchronized (lock) {
    while (!condition) {
        lock.wait();
    }
    lock.notifyAll();
}
```

注意：

- `wait()` 会释放锁。
- `sleep()` 不释放锁。
- `wait/notify` 必须持有对象监视器，否则抛 `IllegalMonitorStateException`。
- `wait()` 会抛 `InterruptedException`。

# 9. 高频主观题答题模板

## 9.1 问：解释面向对象三要素，并说明Java实现方式

答题模板：

> 面向对象三要素是封装、继承和多态。封装是把对象的数据和操作数据的方法绑定在一起，并通过访问控制隐藏内部实现，在Java中通常用private修饰字段，用public方法作为外部接口。继承是子类复用父类属性和方法的机制，在Java中用extends实现，体现is-a关系。多态是同一父类引用或接口引用在运行时根据实际对象表现出不同行为，在Java中通过方法重写、向上转型和动态绑定实现。

---

## 9.2 问：String中 `==` 和 `equals()` 区别

答题模板：

> `==` 比较两个引用是否指向同一个对象；`equals()` 比较对象内容。String重写了equals方法，因此比较字符串内容应使用equals。字符串字面量可能进入字符串常量池，所以两个字面量用==可能为true，但new String创建的是新对象，用==通常为false。

---

## 9.3 问：为什么重写 equals 必须重写 hashCode

答题模板：

> 因为基于哈希的集合如HashSet、HashMap会先使用hashCode定位位置，再用equals判断相等。Java约定：如果两个对象equals为true，它们的hashCode必须相同。若只重写equals不重写hashCode，相等对象可能进入不同哈希桶，导致HashSet去重失败或HashMap取值失败。

---

## 9.4 问：checked exception 和 unchecked exception 区别

答题模板：

> checked exception 是编译器强制处理的异常，方法中可能抛出时必须try-catch或在方法声明处throws，如IOException。unchecked exception 通常指RuntimeException及其子类，编译器不强制处理，如NullPointerException、ArithmeticException，通常代表程序逻辑错误。

---

## 9.5 问：ArrayList、LinkedList、HashSet、TreeSet、HashMap区别

答题模板：

> ArrayList底层类似动态数组，按下标访问快；LinkedList底层是链表，插入删除较方便，也可作队列；HashSet基于哈希表，不允许重复且无序，查询平均较快；TreeSet基于树结构，不允许重复并按规则排序，元素需要可比较；HashMap存储键值对，key不能重复，通过key查value，key应正确实现equals和hashCode。
