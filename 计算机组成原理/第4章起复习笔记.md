# 第 4 章 指令系统

## 4.1 这一章到底学什么？

指令系统这一章主要回答三个问题：

```txt
1. 一条机器指令长什么样？
2. 指令里的地址码怎么找到真正的操作数？
3. RISC-V 这类精简指令集的指令格式和常见指令怎么理解？
```

核心词：

| 术语 | 你要怎么理解 |
|---|---|
| 指令 | CPU 能直接识别和执行的命令 |
| 指令系统 | 一台机器所有机器指令的集合 |
| 操作码 OP | 说明“做什么操作”，比如加、减、取数、存数、跳转 |
| 地址码 A | 说明“操作数在哪里”或“跳到哪里” |
| 寻址方式 | 根据指令中的地址码确定有效地址 EA 的方法 |
| 有效地址 EA | 真正要访问的主存地址 |
| 指令格式 | 操作码、寄存器号、立即数、地址码等字段的排列方式 |

---

## 4.2 指令格式：一条指令由什么组成？

一般格式：

```txt
| 操作码 OP | 寻址方式字段 | 地址码 / 寄存器号 / 立即数 |
```

例如：

```asm
LOAD R1, [1000H]
```

可以理解为：

```txt
OP：LOAD，表示取数
R1：目的寄存器
1000H：地址码
寻址方式：[1000H] 表示直接访问主存 1000H
```

考试常问：

```txt
若指令字长为 16 位，操作码占 4 位，地址码占多少位？
若操作码有 n 位，最多能表示多少条指令？
```

公式：

```txt
n 位操作码最多表示 2^n 条指令
n 位地址码最多表示 2^n 个地址
```

---

## 4.3 操作数寻址 / 数据寻址

你前面问过“数据寻址”和“操作数寻址”区别。考试里基本可以当成一回事：

> **数据寻址 = 操作数寻址 = 找指令要用的数据在哪里。**

它和 **指令寻址** 不一样：

| 类型 | 找什么 |
|---|---|
| 指令寻址 | 下一条指令在哪里 |
| 数据寻址 / 操作数寻址 | 本条指令要用的数据在哪里 |

---

## 4.4 常见寻址方式总表

设：

```txt
A：指令中的地址码
EA：有效地址
M[x]：主存地址 x 中的内容
R[x]：寄存器 x 中的内容
IX：变址寄存器
BR：基址寄存器
PC：程序计数器
```

| 寻址方式 | 操作数在哪里 | 有效地址 EA |
|---|---|---|
| 立即寻址 | 操作数就在指令中 | 无 EA，A 本身就是操作数 |
| 直接寻址 | 操作数在主存 A 中 | EA = A |
| 间接寻址 | A 中存的是操作数地址 | EA = M[A] |
| 寄存器寻址 | 操作数在寄存器 R 中 | 无主存 EA |
| 寄存器间接寻址 | 寄存器中存的是主存地址 | EA = R[i] |
| 变址寻址 | 地址码 + 变址寄存器 | EA = A + IX |
| 基址寻址 | 地址码 + 基址寄存器 | EA = A + BR |
| 相对寻址 | 地址码 + PC | EA = PC + A |

### 最容易混的几个

#### 立即寻址

```asm
MOV R1, #5
```

意思是：

```txt
5 这个数就在指令里
```

不需要访问主存取操作数。

#### 直接寻址

```asm
MOV R1, [1000H]
```

意思是：

```txt
操作数在主存 1000H 单元里
EA = 1000H
```

#### 间接寻址

```asm
MOV R1, [[1000H]]
```

意思是：

```txt
1000H 里面存的不是操作数，而是操作数地址
EA = M[1000H]
操作数 = M[EA]
```

#### 变址寻址

```txt
EA = 地址码 A + 变址寄存器 IX
```

常用于数组。

#### 相对寻址

```txt
EA = PC + 位移量
```

常用于转移指令。

## 4.5 有效地址计算例题

文件中的题目条件：

```txt
当前指令地址码部分 A = 001AH
正在执行的指令所在地址 = 1F05H
变址寄存器 IX = 23A0H

存储器部分内容：
M[001AH] = 23A0H
M[1F05H] = 2400H
M[1F1FH] = 2500H
M[23A0H] = 2600H
M[23BAH] = 1748H
```

### 1）变址寻址

公式：

```txt
EA = A + IX
```

代入：

```txt
EA = 001AH + 23A0H = 23BAH
```

题目问“取出来的数”，所以还要取主存内容：

```txt
M[23BAH] = 1748H
```

答案：

```txt
1748H
```

### 2）间接寻址

公式：

```txt
EA = M[A]
```

代入：

```txt
EA = M[001AH] = 23A0H
```

题目问“取出来的数”，所以：

```txt
M[23A0H] = 2600H
```

答案：

```txt
2600H
```

### 3）相对寻址

公式：

```txt
EA = PC + A
```

这题给的是“正在执行的指令所在地址为 1F05H”。若按题目给定当前地址直接计算：

```txt
EA = 1F05H + 001AH = 1F1FH
```

答案：

```txt
EA = 1F1FH
```

注意：有些教材把取指后的 PC 视为下一条指令地址。如果按“取指后 PC 已经加 1”处理，可能会用 `1F06H + 001AH = 1F20H`。但这道题表格特意给了 `1F1FH`，所以更可能希望你按当前指令地址 `1F05H` 算。

## 4.6 CISC 和 RISC

| 对比项 | CISC | RISC |
|---|---|---|
| 名称 | 复杂指令集计算机 | 精简指令集计算机 |
| 指令数量 | 多 | 少 |
| 指令长度 | 常不固定 | 通常固定 |
| 指令功能 | 单条指令可做复杂操作 | 单条指令功能简单 |
| 控制器 | 常用微程序控制 | 更适合硬布线控制 |
| 执行周期 | 可能多个周期 | 多数简单指令适合流水线 |
| 代表 | x86 | RISC-V、MIPS、ARM 部分架构 |

考试答法：

```txt
CISC 指令数量多、格式复杂、寻址方式多，单条指令功能强，但控制复杂。
RISC 指令数量少、格式规整、长度固定，访存指令较少，便于流水线和硬布线控制。
```

## 4.7 RISC-V 六种基本指令格式

常见 RISC-V 基本格式：

| 格式 | 常见指令 | 用途 |
|---|---|---|
| R 型 | add, sub, and, or, slt | 寄存器-寄存器运算 |
| I 型 | addi, lw, jalr | 立即数运算、取数 |
| S 型 | sw | 存数 |
| B 型 | beq, bne | 条件分支 |
| U 型 | lui, auipc | 高位立即数 |
| J 型 | jal | 无条件跳转并链接 |

### R 型

```asm
add rd, rs1, rs2
```

含义：

```txt
R[rd] ← R[rs1] + R[rs2]
```

### I 型

```asm
addi rd, rs1, imm
```

含义：

```txt
R[rd] ← R[rs1] + SignExt(imm)
```

取数：

```asm
lw rd, imm(rs1)
```

含义：

```txt
R[rd] ← M[R[rs1] + SignExt(imm)]
```

### S 型

```asm
sw rs2, imm(rs1)
```

含义：

```txt
M[R[rs1] + SignExt(imm)] ← R[rs2]
```

### B 型

```asm
beq rs1, rs2, label
```

含义：

```txt
if R[rs1] == R[rs2]:
    PC ← PC + offset
else:
    PC ← PC + 4
```

### U 型

```asm
lui rd, imm
```

含义：

```txt
R[rd] ← imm << 12
```

### J 型

```asm
jal rd, label
```

含义：

```txt
R[rd] ← PC + 4
PC ← PC + offset
```

## 4.8 RTL 描述怎么写？

RTL 是 Register Transfer Language，寄存器传送语言。

它不是代码，而是描述数据怎么在寄存器、ALU、存储器之间流动。

常见写法：

```txt
R1 ← R2
R3 ← R1 + R2
MAR ← PC
MDR ← M[MAR]
IR ← MDR
PC ← PC + 4
```

### 常见指令 RTL

#### LOAD

```txt
MAR ← addr
MDR ← M[MAR]
R[rd] ← MDR
```

#### STORE

```txt
MAR ← addr
MDR ← R[rs]
M[MAR] ← MDR
```

#### ADD

```txt
R[rd] ← R[rs1] + R[rs2]
```

#### BEQ

```txt
if R[rs1] == R[rs2]:
    PC ← PC + offset
else:
    PC ← PC + 4
```

## 5.2 CPU 基本组成

| 部件 | 作用 |
|---|---|
| 运算器 ALU | 执行加减、逻辑、比较、移位等操作 |
| 控制器 CU | 产生控制信号，指挥数据通路 |
| 寄存器组 | 暂存操作数和结果 |
| PC | 程序计数器，保存下一条指令地址 |
| IR | 指令寄存器，保存当前指令 |
| MAR | 主存地址寄存器，保存要访问的主存地址 |
| MDR | 主存数据寄存器，保存从主存读出或要写入主存的数据 |
| 状态寄存器 | 保存零标志、符号标志、进位、溢出等状态 |

---

## 5.3 指令周期

一条指令通常经过：

```txt
取指 → 译码 → 执行 → 访存 → 写回
```

对应流水线常见五段：

| 阶段 | 英文 | 作用 |
|---|---|---|
| 取指 | IF | 根据 PC 从指令存储器取指 |
| 译码/读寄存器 | ID | 分析指令，读寄存器 |
| 执行/计算地址 | EX | ALU 运算或计算有效地址 |
| 访存 | MEM/M | 读写数据存储器 |
| 写回 | WB | 把结果写回寄存器 |

---

## 5.4 传统 CPU 的取指微操作

通用取指流程：

```txt
MAR ← PC
MDR ← M[MAR]
IR ← MDR
PC ← PC + 1      // 若按字编址且每条指令一字
```

如果是 RISC-V 这种 32 位定长指令，PC 通常加 4：

```txt
IR ← M[PC]
PC ← PC + 4
```

## 5.5 文件中的 CPU 结构图题：a、b、c、d 是什么？

## 5.6 LDA addr 的数据通路

题目功能：

```txt
LDA addr：把主存 addr 单元的内容送入 AC
```

意思是：

```txt
AC ← M[addr]
```

完整过程可以拆成取指 + 执行。

### 取指阶段

```txt
MAR ← PC
MDR ← M[MAR]
IR ← MDR
PC ← PC + 1
```

### 执行阶段

```txt
MAR ← addr(IR)
MDR ← M[MAR]
AC ← MDR
```

写成一句：

```txt
PC 给出指令地址，取出 LDA 指令送 IR；
IR 中的地址码 addr 送 MAR；
主存读出 M[addr] 到 MDR；
MDR 的内容送 AC。
```

## 5.7 单周期 CPU 是什么？

单周期 CPU：

> 每条指令都在一个时钟周期内完成。

特点：

```txt
优点：控制简单
缺点：时钟周期必须按最慢指令设计，效率低
```

在单周期 CPU 中，所有指令共用一套数据通路，但控制信号不同。

---

## 5.8 单周期 RISC-V 数据通路核心部件

常见部件：

```txt
PC
指令存储器 Instruction Memory
寄存器堆 Register File
立即数生成器 ImmGen
ALU
数据存储器 Data Memory
加法器 PC+4
分支目标地址加法器
多路选择器 MUX
控制器 Control Unit
ALU 控制器 ALU Control
```

你要理解：

```txt
不同指令就是让数据经过不同路径。
控制信号就是用来打开/选择这些路径。
```

## 5.9 单周期控制信号总表

常见控制信号：

| 控制信号 | 作用 |
|---|---|
| RegWrite | 是否写寄存器 |
| ALUSrc | ALU 第二个操作数来自寄存器还是立即数 |
| MemRead | 是否读数据存储器 |
| MemWrite | 是否写数据存储器 |
| MemtoReg | 写回寄存器的数据来自 ALU 还是内存 |
| Branch | 是否为条件分支 |
| Jump | 是否跳转 |
| ALUOp | 指示 ALU 做什么类型操作 |
| ImmSrc | 立即数类型选择 |

## 5.10 常见指令控制信号

| 指令类型 | RegWrite | ALUSrc | MemRead | MemWrite | MemtoReg | Branch | Jump | ALU 主要操作 |
|---|---:|---:|---:|---:|---:|---:|---:|---|
| R 型 add/sub | 1 | 0 | 0 | 0 | 0 | 0 | 0 | 加/减/逻辑 |
| I 型 addi | 1 | 1 | 0 | 0 | 0 | 0 | 0 | 加 |
| lw | 1 | 1 | 1 | 0 | 1 | 0 | 0 | 地址加法 |
| sw | 0 | 1 | 0 | 1 | X | 0 | 0 | 地址加法 |
| beq | 0 | 0 | 0 | 0 | X | 1 | 0 | 减法/比较 |
| jal | 1 | X | 0 | 0 | PC+4 | 0 | 1 | 计算跳转 |
| lui | 1 | X | 0 | 0 | 立即数 | 0 | 0 | 立即数写回 |

说明：

```txt
X 表示不关心。
不同教材信号名可能略有不同，但判断逻辑一样。
```

## 5.11 各类指令的数据通路

### R 型指令：add rd, rs1, rs2

功能：

```txt
R[rd] ← R[rs1] + R[rs2]
```

路径：

```txt
PC → 指令存储器 → IR
寄存器堆读 rs1、rs2
ALU 执行加法
ALU 结果写回 rd
PC ← PC + 4
```

特点：

```txt
不访问数据存储器
写寄存器
ALU 第二操作数来自寄存器
```

### I 型立即数指令：addi rd, rs1, imm

功能：

```txt
R[rd] ← R[rs1] + imm
```

路径：

```txt
读 rs1
立即数扩展
ALU 加法
结果写回 rd
PC ← PC + 4
```

特点：

```txt
ALU 第二操作数来自立即数
写寄存器
不访存
```

### lw 指令

功能：

```txt
R[rd] ← M[R[rs1] + imm]
```

路径：

```txt
读 rs1
立即数扩展
ALU 计算有效地址
数据存储器读
读出的数据写回 rd
```

特点：

```txt
读内存
写寄存器
写回数据来自内存
```

### sw 指令

功能：

```txt
M[R[rs1] + imm] ← R[rs2]
```

路径：

```txt
读 rs1 和 rs2
立即数扩展
ALU 计算有效地址
把 rs2 的值写入数据存储器
```

特点：

```txt
写内存
不写寄存器
```

### beq 指令

功能：

```txt
if R[rs1] == R[rs2]:
    PC ← PC + offset
else:
    PC ← PC + 4
```

路径：

```txt
读 rs1、rs2
ALU 做减法或比较
若相等，选择分支目标地址
否则 PC+4
```

特点：

```txt
不写寄存器
不读写数据存储器
改变 PC
```

### jal 指令

功能：

```txt
R[rd] ← PC + 4
PC ← PC + offset
```

特点：

```txt
写回 PC+4
同时改变 PC
```

## 5.12 关键路径是什么？

关键路径：

> 一条指令执行时，数据经过的最长组合逻辑路径。

单周期 CPU 的时钟周期必须大于最慢指令的关键路径时间。

通常最慢的是 load：

```txt
取指令存储器
读寄存器
ALU 算地址
读数据存储器
写回寄存器
```

所以：

```txt
lw 往往决定单周期 CPU 的时钟周期。
```

---

## 5.13 多周期 CPU

多周期 CPU：

> 一条指令分多个时钟周期完成，不同阶段复用硬件。

特点：

```txt
每个周期只做一小步
不同指令周期数可以不同
硬件利用率比单周期高
控制比单周期复杂
```

常见阶段：

```txt
IF：取指
ID：译码和读寄存器
EX：执行或计算地址
MEM：访存
WB：写回
```

和流水线不同：

```txt
多周期 CPU 同一时间通常只执行一条指令的一个阶段。
流水线 CPU 同一时间可以让多条指令处在不同阶段。
```

---

## 5.14 微程序控制器

微程序控制的思想：

> 把控制信号也看成一种“微指令”，存放在控制存储器中。

术语：

| 术语 | 含义 |
|---|---|
| 微命令 | 最小的控制动作，如 PCout、MARin、Read |
| 微操作 | 微命令控制下完成的数据传送或运算 |
| 微指令 | 一组微命令的编码 |
| 微程序 | 完成一条机器指令的一串微指令 |
| 控制存储器 | 存放微程序的 ROM |
| 微地址 | 微指令在控制存储器中的地址 |

和硬布线控制对比：

| 对比 | 硬布线控制 | 微程序控制 |
|---|---|---|
| 速度 | 快 | 慢一些 |
| 修改 | 难 | 较容易 |
| 适合 | RISC | CISC |
| 实现 | 逻辑电路 | 控制存储器 + 微指令 |

---

## 5.15 流水线 CPU

五段流水线：

```txt
IF → ID → EX → MEM → WB
```

如果有 `n` 条指令，流水线级数为 `k`，没有阻塞时：

```txt
总周期数 = k + n - 1
```

例如文件中说连续 4 条指令进入 5 段流水线，无阻塞：

```txt
总周期数 = 5 + 4 - 1 = 8
```

---

## 5.16 流水线冒险

冒险就是：

> 下一条指令不能按理想流水线继续执行。

分三类：

| 冒险类型 | 原因 | 解决办法 |
|---|---|---|
| 结构冒险 | 多条指令争用同一硬件资源 | 增加硬件、分离指令/数据 Cache、暂停 |
| 数据冒险 | 后一条指令需要前一条尚未写回的数据 | 转发、暂停、编译器调度 |
| 控制冒险 | 分支/跳转导致下一条指令地址不确定 | 分支预测、延迟槽、暂停、冲刷流水线 |

---

## 5.17 数据冒险：RAW 最常考

RAW：Read After Write，写后读相关。

例子：

```asm
I1: LOAD R1, [a]
I2: ADD R1, R2
```

I2 要读 R1，但 I1 还没把内存数据写回 R1。

如果没有转发技术，I2 必须等 I1 到 WB 之后才能在 ID 阶段读寄存器。

文件里还特别说明：

```txt
没有采用转发技术
同一个寄存器的读和写操作不能在同一个时钟周期内进行
```

这意味着：

```txt
若某条指令在第 t 个周期 WB 写寄存器，
使用它的后继指令 ID 读寄存器至少要在第 t+1 个周期。
```

这比“同周期前半写后半读”的情况更严格。

## 5.18 文件中的流水线题关键点

指令表：

```asm
ADD Rs, Rd      (Rs) + (Rd) → Rd
SHL Rd          2 * (Rd) → Rd
SHR Rd          (Rd) / 2 → Rd
LOAD Rd, mem    (mem) → Rd
STORE Rs, mem   (Rs) → mem
```

### 1）SHL R1 例题

题目：

```txt
16 位计算机，带符号整数用补码表示。
int x = -513，存放在 R1 中。
执行 SHL R1 后，R1 内容是多少？用十六进制表示。
```

16 位补码：

```txt
+513 = 0201H
-513 = 10000H - 0201H = FDFFH
```

SHL 左移一位，相当于乘 2，低位补 0：

```txt
FDFFH << 1 = FBFEH
```

所以：

```txt
R1 = FBFEH
```

它对应真值：

```txt
-1026
```

注意：左移后只保留 16 位。

### 2）4 条指令无阻塞周期数

5 段流水线，4 条指令：

```txt
周期数 = 5 + 4 - 1 = 8
```

### 3）I3 的 ID 段为什么阻塞？

文件给的指令序列：

```asm
I1 LOAD R1, [a]
I2 LOAD R2, [b]
I3 ADD R1, R2
I4 STORE R2, [x]
```

I3 要读 R1 和 R2。

但是：

```txt
R1 由 I1 的 LOAD 写回
R2 由 I2 的 LOAD 写回
```

没有转发技术，而且读写同一寄存器不能同周期，所以 I3 在 ID 阶段必须等 I1、I2 写回完成后才能读寄存器。

答案写法：

```txt
I3 的 ID 段被阻塞，是因为 I3 与前面的 I1、I2 存在 RAW 数据相关。
I3 需要读取 R1、R2，但这两个寄存器分别由 I1、I2 的 LOAD 指令产生；
由于没有转发技术，且寄存器读写不能在同一周期进行，所以 I3 必须等待前面指令写回后才能继续。
```

### 4）I4 的 IF 段为什么阻塞？

I3 卡在 ID 阶段时，后续指令不能按序进入流水线。

答案写法：

```txt
I4 的 IF 段被阻塞，是因为流水线按序发射，I3 阻塞在 ID 段，导致后续 I4 不能进入译码阶段，取指阶段也被迫暂停。
```

## 5.19 做流水线图的通用方法

遇到流水线画图题：

```txt
1. 先写出指令序列
2. 默认按 IF、ID、EX、M、WB 斜着排
3. 检查每条指令的 ID 阶段是否需要读前面尚未写回的寄存器
4. 若没有转发，就让 ID 阶段等待到生产者 WB 之后
5. 若题目说同周期不能读写同一寄存器，则消费者 ID 必须在生产者 WB 的下一周期
6. ID 卡住时，后续 IF 也卡住
```

## 7.2 存储器层次结构

从快到慢：

```txt
寄存器 → Cache → 主存 → 外存
```

| 层次 | 特点 |
|---|---|
| 寄存器 | 最快、容量最小、最贵 |
| Cache | 很快，缓存主存数据 |
| 主存 | CPU 可直接访问，容量较大 |
| 外存 | 容量最大，速度慢，断电不丢失 |

核心思想：

```txt
利用局部性原理，用小而快的存储器缓存大而慢的存储器中的常用内容。
```

局部性：

| 类型 | 含义 |
|---|---|
| 时间局部性 | 刚访问过的数据近期可能再次访问 |
| 空间局部性 | 访问某地址后，附近地址也可能被访问 |

## 7.3 SRAM 和 DRAM

| 对比 | SRAM | DRAM |
|---|---|---|
| 存储元件 | 触发器 | 电容 |
| 是否刷新 | 不需要 | 需要 |
| 速度 | 快 | 较慢 |
| 集成度 | 低 | 高 |
| 成本 | 高 | 低 |
| 常见用途 | Cache | 主存 |

考试句：

```txt
Cache 通常用 SRAM，主存通常用 DRAM。
```

## 7.4 地址线、数据线、地址寄存器、数据寄存器

### 芯片写法：m × n 位

例如：

```txt
64K × 16 位
```

意思是：

```txt
有 64K 个地址单元
每个地址单元 16 位
```

所以：

```txt
地址线数 = log2(64K) = 16 根
数据线数 = 16 根
```

记法：

```txt
m × n 位：
m 决定地址线
n 决定数据线
```

### 给总容量时

例如：

```txt
32KB 容量，按 16 位字编址
```

这时要先算地址个数：

```txt
16 位字 = 2B
地址个数 = 32KB / 2B = 16K = 2^14
地址寄存器 = 14 位
数据寄存器 = 16 位
```

## 7.5 位扩展、字扩展、字位同时扩展

### 位扩展

目标数据位数变大，地址数量不变。

例：

```txt
用 8K×4 芯片组成 8K×8 存储器
```

需要：

```txt
2 片并联
一片接 D0~D3
一片接 D4~D7
```

### 字扩展

目标地址数量变大，数据位数不变。

例：

```txt
用 8K×8 芯片组成 32K×8 存储器
```

需要：

```txt
32K / 8K = 4 片
```

用高位地址线片选。

### 字位同时扩展

例：

```txt
用 16K×4 芯片组成 64K×8 存储器
```

先位扩展：

```txt
4 位 → 8 位，需要 2 片一组
```

再字扩展：

```txt
16K → 64K，需要 4 组
```

总片数：

```txt
2 × 4 = 8 片
```

## 7.6 主存与 CPU 连接图怎么想？

连接图只分三类线：

```txt
地址线：CPU 告诉存储器访问哪里
数据线：CPU 和存储器传数据
控制/片选线：决定哪片芯片工作、读还是写
```

规律：

```txt
低位地址线 → 接芯片内部地址端
高位地址线 → 经过译码器产生片选信号 CS
数据线 → 接 CPU 数据总线
读写控制线 → 接 OE/WE 或 RD/WR
```

## 7.8 DRAM 刷新题

### 为什么 DRAM 要刷新？

DRAM 用电容存数据，电容会漏电，所以必须周期性刷新。

### 为什么按行刷新？

DRAM 组织成行列矩阵。

```txt
先选一行，再选列。
刷新时打开某一行，这一整行都会被恢复电荷。
```

所以刷新单位是：

```txt
行
```

不是列。

### 行数怎么算？

例如：

```txt
64K×1 DRAM
```

容量：

```txt
64K = 2^16
```

通常按接近平方矩阵组织：

```txt
2^16 = 2^8 × 2^8
```

所以：

```txt
行数 = 2^8 = 256 行
```

再如：

```txt
16K×1 = 2^14 = 2^7 × 2^7
```

所以：

```txt
行数 = 128 行
```

---

## 7.9 DRAM 刷新例题：64K×1 构成 1M×8

题目：

```txt
用 64K×1 位 DRAM 芯片构成 1M×8 位存储器。
若采用异步刷新，每行刷新间隔不超过 2ms，
则产生刷新信号的间隔是多少？
读写周期为 0.5us，若采用集中刷新方式，
刷新一遍最少用多少个读写周期？CPU 死时间为多少？
```

### 第一步：算单片行数

```txt
64K = 2^16 = 2^8 × 2^8
```

所以：

```txt
行数 = 256 行
```

### 异步刷新信号间隔

```txt
每行刷新间隔不超过 2ms
```

2ms 内要把 256 行都刷新一次：

```txt
刷新信号间隔 = 2ms / 256 = 2000us / 256 = 7.8125us
```

### 集中刷新

集中刷新一次刷完整个芯片所有行：

```txt
需要 256 个读写周期
```

读写周期：

```txt
0.5us
```

CPU 死时间：

```txt
256 × 0.5us = 128us
```

注意：

```txt
1M×8 主要用于算芯片片数；刷新计算看单片 DRAM 的行数。
```

## 7.10 Cache 基本原理

Cache 是 CPU 和主存之间的高速缓冲。

CPU 访存时：

```txt
先查 Cache
命中 → 直接从 Cache 取
未命中 → 到主存取，并把主存块调入 Cache
```

核心术语：

| 术语 | 含义 |
|---|---|
| Cache 块 | Cache 和主存交换的基本单位 |
| 主存块 | 主存按块划分后的单位 |
| 命中 | CPU 要的数据在 Cache 中 |
| 未命中 | CPU 要的数据不在 Cache 中 |
| 命中率 H | 命中次数 / 总访问次数 |

---

## 7.11 Cache 地址映射

### 直接映射

主存块只能放到 Cache 的固定块中。

公式：

```txt
Cache 块号 = 主存块号 mod Cache 块数
```

主存地址格式：

```txt
| 标记 Tag | Cache 块号 | 块内地址 |
```

优点：

```txt
硬件简单，查找快
```

缺点：

```txt
冲突多
```

### 全相联映射

主存块可以放到 Cache 任意块中。

主存地址格式：

```txt
| 标记 Tag | 块内地址 |
```

优点：

```txt
冲突少
```

缺点：

```txt
比较电路复杂，要和所有 Cache 块标记比较
```

### 组相联映射

折中方式。

主存块先映射到某一组，再放入该组任意一路。

主存地址格式：

```txt
| 标记 Tag | 组号 | 块内地址 |
```

公式：

```txt
组号 = 主存块号 mod Cache 组数
```

---

## 7.12 Cache 地址划分模板

给定：

```txt
主存块数 = 2^m
Cache 块数 = 2^c
每块大小 = 2^b 个字/字节
若为 k 路组相联，则 Cache 组数 = Cache 块数 / k = 2^s
```

### 直接映射

```txt
主存地址 = Tag + Cache块号 + 块内地址

块内地址位数 = b
Cache块号位数 = c
Tag 位数 = m - c
```

### 全相联

```txt
主存地址 = Tag + 块内地址

块内地址位数 = b
Tag 位数 = m
```

### k 路组相联

```txt
主存地址 = Tag + 组号 + 块内地址

块内地址位数 = b
组号位数 = s
Tag 位数 = m - s
```

## 7.13 Cache 地址划分例题

题目：

```txt
Cache 有 64 个存储块
主存有 4096 个存储块
每块 128 个字
访问地址为字地址
```

基础量：

```txt
Cache 块数 = 64 = 2^6
主存块数 = 4096 = 2^12
每块字数 = 128 = 2^7
```

所以：

```txt
主存地址总位数 = 12 + 7 = 19 位
Cache 地址总位数 = 6 + 7 = 13 位
```

```txt
Tag = 12 - 6 = 6 位
Cache 块号 = 6 位
块内地址 = 7 位
```

格式：

```txt
| Tag 6 | Cache块号 6 | 块内地址 7 |
```

### 全相联

```txt
Tag = 12 位
块内地址 = 7 位
```

格式：

```txt
| Tag 12 | 块内地址 7 |
```

### 四路组相联

四路表示每组 4 块。

```txt
组数 = 64 / 4 = 16 = 2^4
组号 = 4 位
Tag = 12 - 4 = 8 位
块内地址 = 7 位
```

格式：

```txt
| Tag 8 | 组号 4 | 块内地址 7 |
```

## 7.14 Cache 命中率例题

题目：

```txt
主存容量 4MB，Cache 容量 16KB。
每块含 8 个字，每字 32 位。
全相联映射，Cache 初始为空。
CPU 依次读主存第 0,1,2,...,99 号单元，每次读一个字，并重复 10 次。
替换算法 LRU。
求 Cache 命中率。
```

### 从公式出发

命中率公式：

```txt
H = 命中次数 / 总访问次数
```

也可以写成：

```txt
H = (总访问次数 - 未命中次数) / 总访问次数
```

所以要算：

```txt
1. 总访问次数
2. 未命中次数
```

### 总访问次数

一次读：

```txt
0 ~ 99，共 100 个字
```

重复 10 次：

```txt
总访问次数 = 100 × 10 = 1000
```

### 未命中次数

每块 8 个字，所以：

```txt
第 0 块：0~7
第 1 块：8~15
第 2 块：16~23
...
```

访问 0~99 共 100 个字，涉及：

```txt
ceil(100 / 8) = 13 个主存块
```

Cache 初始为空，每个新块第一次访问不命中一次，所以第一次循环有：

```txt
13 次未命中
```

Cache 块数：

```txt
每字 32 位 = 4B
每块 = 8 × 4B = 32B
Cache 容量 = 16KB
Cache 块数 = 16KB / 32B = 512 块
```

Cache 能装下这 13 个块，后面 9 次重复全部命中。

所以：

```txt
总未命中次数 = 13
命中次数 = 1000 - 13 = 987
```

命中率：

```txt
H = 987 / 1000 = 98.7%
```

## 7.15 Cache 平均访问时间和速度提高

设：

```txt
Tc：Cache 访问时间
Tm：主存访问时间
H：命中率
r = Tm / Tc
```

若按常见公式：

```txt
Ta = H × Tc + (1 - H) × Tm
```

代入 `Tm = rTc`：

```txt
Ta = [H + (1 - H)r]Tc
```

如果题目说：

```txt
Cache 比主存快 10 倍
```

说明：

```txt
r = Tm/Tc = 10
```

用上题命中率：

```txt
H = 0.987
1-H = 0.013
```

平均访问时间：

```txt
Ta = [0.987 + 0.013 × 10]Tc
   = 1.117Tc
```

没有 Cache 时访问时间：

```txt
Tm = 10Tc
```

速度提高到原来的：

```txt
10Tc / 1.117Tc ≈ 8.95 倍
```

## 7.16 替换算法 FIFO 和 LRU

### FIFO

First In First Out，先进先出。

```txt
最早进入 Cache 的块先被替换。
```

不管它最近是否常用。

### LRU

Least Recently Used，最近最少使用。

```txt
替换最长时间没有被访问的块。
```

更符合时间局部性，但实现复杂。

### 写直达 Write Through

```txt
写 Cache 的同时也写主存
```

优点：

```txt
主存和 Cache 一致
```

缺点：

```txt
写操作慢
```

```txt

```

优点：

```txt
写操作快，减少主存访问
```

缺点：

```txt
一致性控制复杂
```

## 7.18 虚拟存储器

虚拟存储器让程序看到一个比实际主存更大的地址空间。

核心：

```txt
虚拟地址 VA → 地址变换 → 物理地址 PA
```

分页系统中：

```txt
虚拟地址 = VPN + VPO
物理地址 = PPN + PPO
```

| 字段 | 含义 |
|---|---|
| VPN | 虚拟页号 |
| VPO | 页内偏移 |
| PPN | 物理页号 |
| PPO | 页内偏移 |

页内偏移不变：

```txt
VPO = PPO
```

因为虚拟页和物理页大小相同。

## 7.19 虚存地址划分例题

题目：

```txt
主存容量 8MB
虚存容量 2GB
页面大小 4KB
求 VPN、VPO、PPN、PPO 位数
```

换成 2 的幂：

```txt
主存 8MB = 2^23 B
虚存 2GB = 2^31 B
页面 4KB = 2^12 B
```

页内偏移：

```txt
VPO = PPO = 12 位
```

虚拟地址总位数：

```txt
2GB = 2^31，所以虚拟地址 31 位
```

物理地址总位数：

```txt
8MB = 2^23，所以物理地址 23 位
```

所以：

```txt
VPN = 31 - 12 = 19 位
PPN = 23 - 12 = 11 位
```

答案：

```txt
VPN = 19 位
VPO = 12 位
PPN = 11 位
PPO = 12 位
```

---

## 7.20 TLB 是什么？

TLB：Translation Lookaside Buffer，快表。

作用：

```txt
缓存页表项，加快虚拟地址到物理地址的转换。
```

访问流程：

```txt
CPU 给出虚拟地址
先查 TLB
TLB 命中 → 直接得到物理页号 PPN
TLB 未命中 → 查页表
```

TLB 本质上也是一种 Cache，只不过缓存的是：

```txt
虚拟页号 VPN 到物理页号 PPN 的映射关系
```

## 7.21 磁盘基础

磁盘基本结构：

| 术语 | 含义 |
|---|---|
| 盘片 | 磁盘中的圆形存储介质 |
| 磁道 | 盘面上的同心圆 |
| 扇区 | 磁道被划分的小块 |
| 柱面 | 多个盘面上半径相同的磁道集合 |
| 磁头 | 读写数据的部件 |

磁盘平均访问时间：

```txt
平均存取时间 = 平均寻道时间 + 平均旋转延迟 + 数据传输时间
```

平均旋转延迟：

```txt
平均旋转延迟 = 旋转一圈时间 / 2
```

# 第 8/9 章 总线互连及输入输出组织

## 8.1 总线是什么？

总线：

> 计算机各部件之间传送信息的一组公共通信线路。

总线传输的信息包括：

```txt
地址
数据
控制信号
```

---

## 8.2 总线的 4 个特征

| 特征 | 含义 |
|---|---|
| 机械特性 | 插头、引脚、尺寸等物理结构 |
| 电气特性 | 电平、信号方向、负载能力 |
| 功能特性 | 各条线的功能，如地址线、数据线、控制线 |
| 时间特性 | 信号时序、何时有效、保持多久 |

---

## 8.3 总线按功能分类

| 总线 | 作用 |
|---|---|
| 地址总线 | 传送地址 |
| 数据总线 | 传送数据 |
| 控制总线 | 传送读写、中断、时钟等控制信号 |

特点：

```txt
地址总线通常单向：CPU → 存储器/I/O
数据总线通常双向
控制总线方向视具体信号而定
```

---

## 8.4 信息传输方式

文件中提到四种：

| 方式 | 含义 |
|---|---|
| 并行传送 | 多位数据同时传送，速度快，线多 |
| 串行传送 | 数据一位一位传送，线少，速度相对慢 |
| 分时传送 | 同一组线在不同时间传送不同信息 |
| 突发传送 | 给出首地址后连续传送多个数据 |

---

## 8.5 集中式总线仲裁三种方式

总线仲裁：

> 多个设备都想用总线时，决定谁先用。

### 链式查询

优点：

```txt
结构简单，线少
```

缺点：

```txt
优先级固定，靠前设备优先级高，后面设备可能饥饿
```

### 计数器查询

思想：

```txt
用计数器轮流查询设备
```

优点：

```txt
优先级可变，比较公平
```

### 独立请求

每个设备都有独立请求线。

优点：

```txt
响应快，优先级灵活
```

缺点：

```txt
控制线多，硬件复杂
```

---

## 9.1 I/O 数据传送控制方式

常见方式：

| 方式 | 特点 |
|---|---|
| 程序查询方式 | CPU 不断查询设备状态，CPU 占用高 |
| 程序中断方式 | 设备准备好后中断 CPU，CPU 再处理 |
| DMA 方式 | 外设和主存直接交换数据，CPU 只负责初始化和结束处理 |

---

## 9.2 程序查询方式

流程：

```txt
CPU 查询设备状态
若未准备好，继续查询
若准备好，执行数据传送
```

缺点：

```txt
CPU 一直等待，效率低
```

---

## 9.3 中断方式

中断流程：

```txt
1. 外设发出中断请求
2. CPU 在适当时刻响应中断
3. 保护现场
4. 转入中断服务程序
5. 执行数据传送或处理
6. 恢复现场
7. 返回原程序
```

中断相关概念：

| 术语 | 含义 |
|---|---|
| 中断请求 | 外设向 CPU 请求服务 |
| 中断响应 | CPU 接受请求 |
| 中断服务程序 | 处理中断的程序 |
| 中断屏蔽 | 暂时禁止某些中断 |
| 中断优先级 | 多个中断同时来时先处理谁 |

---

## 9.4 DMA 方式

DMA：Direct Memory Access，直接存储器访问。

作用：

```txt
外设和主存直接交换数据，不需要 CPU 每个字节都参与。
```

流程：

```txt
1. CPU 初始化 DMA 控制器，设置主存地址、数据长度、传送方向
2. DMA 控制器向 CPU 请求总线
3. CPU 让出总线
4. DMA 控制器直接在外设和主存之间传送数据
5. 传送完成后，DMA 向 CPU 发中断
```

---

## 9.5 DMA 与 CPU 同时使用主存的三种方式

### 停止 CPU 访问主存

DMA 传送期间 CPU 暂停访存。

```txt
优点：控制简单
缺点：CPU 停顿时间长
```

### 周期挪用 / 周期窃取

DMA 每次传送一个字时占用一个主存周期。

```txt
优点：CPU 只被短暂打断
缺点：CPU 运行速度受影响
```

### 交替访问 / 透明 DMA

CPU 和 DMA 交替使用主存，或 DMA 利用 CPU 不访存的周期。

```txt
优点：对 CPU 影响小
缺点：控制复杂
```

---

## 9.6 中断和 DMA 对比

| 对比 | 中断方式 | DMA 方式 |
|---|---|---|
| CPU 是否参与每个数据传送 | 参与 | 不参与 |
| 适合 | 少量、低速数据 | 大批量、高速数据 |
| 数据路径 | 外设 → CPU → 主存 | 外设 ↔ 主存 |
| CPU 开销 | 较大 | 较小 |
| 结束通知 | 中断 | 中断 |

---

## 9.7 CPU 占用率计算思路

题目问：

```txt
采用中断、DMA 方式传送时，CPU 占用率是多少？
```

一般公式：

```txt
CPU 占用率 = CPU 用于 I/O 的时间 / 总时间 × 100%
```

中断方式：

```txt
CPU 用于 I/O 的时间 = 中断次数 × 每次中断服务时间
```

DMA 方式：

```txt
CPU 用于 I/O 的时间 = 初始化 DMA 时间 + 结束中断处理时间
```

若 DMA 周期窃取还要考虑：

```txt
DMA 占用主存周期数 × 每个周期时间
```

---

## 9.8 异步串行通信计算

文件题目：

```txt
采用异步通信方式传送 ASCII 码。
数据位 8 位，校验位 1 位，停止位 1 位。
波特率 4800。
问：
1. 字符传送速率是多少？
2. 每个数据位时间长度是多少？
3. 数据位传送速率是多少？
```

异步串行通信通常还有：

```txt
1 位起始位
```

所以每个字符总传送位数：

```txt
1 起始位 + 8 数据位 + 1 校验位 + 1 停止位 = 11 位
```

### 字符传送速率

```txt
字符速率 = 波特率 / 每字符总位数
```

代入：

```txt
4800 / 11 ≈ 436.36 字符/s
```

### 每个数据位时间长度

波特率 4800 表示每秒传 4800 位。

```txt
每位时间 = 1 / 4800 s ≈ 0.0002083 s = 208.3 us
```

### 有效数据位传送速率

每个字符只有 8 位是真正数据。

```txt
有效数据速率 = 字符速率 × 8
           = 436.36 × 8
           ≈ 3490.9 bit/s
```

也可以：

```txt
有效数据速率 = 4800 × 8 / 11 ≈ 3490.9 bit/s
```

# 最后：最低限度必背公式表

```txt
地址线根数 = log2(地址单元数)

芯片 m×n：
地址线 = log2(m)
数据线 = n

位扩展片数 = 目标位宽 / 单片位宽
字扩展片数 = 目标地址数 / 单片地址数
总片数 = 位扩展片数 × 字扩展片数

Cache 命中率：
H = 命中次数 / 总访问次数

平均访问时间：
Ta = H×Tc + (1-H)×Tm

速度提高倍数：
无 Cache 访问时间 / 有 Cache 平均访问时间

DRAM 刷新间隔：
刷新信号周期 = 最大刷新间隔 / 行数

集中刷新死时间：
死时间 = 行数 × 读写周期

流水线无阻塞周期：
周期数 = 指令条数 + 流水段数 - 1

虚存地址划分：
VPO = PPO = log2(页面大小)
VPN = 虚拟地址位数 - VPO
PPN = 物理地址位数 - PPO
```
